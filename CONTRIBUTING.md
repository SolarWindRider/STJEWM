# How to contribute

This repo is a research codebase. We welcome bug reports, clarifications,
and pull requests that improve reproducibility or fix correctness issues.

## Style

- Python: type hints, no unused imports. `code/core/` and `code/train/` are
  the model + training code; **do not** change these without an associated
  paper-text change.
- Shell scripts: posix-compatible bash. Use `set -e`, no `set -u` (we have
  unset variables in some defaults). Quote all paths with spaces.
- Markdown: keep tables narrow; use fenced code blocks.

## Reproducing a result

Each result in `paper/paper.md` is sourced from a table in
`results/aggregate/`. Each table is regenerated by a script in
`code/scripts/`. Before opening a PR that changes a number, ensure the
relevant script still produces the same table.

## Adding a new env

Add the env class to `code/core/envs/`, register a loader in
`code/data/loaders.py`, add a `make_env` branch in
`code/eval/closed_loop.py`. Test with `code/core/envs/<your_env>.py` smoke
test (should run in <30s).

## Adding a new readout mode

Modify `ReadoutMode` enum in `code/stjewm.py`, add a branch in
`_readout()`, and add a test in the `__main__` smoke block. Run all
`code/scripts/eval_*.sh` to update aggregate tables.

## Reporting bugs

File an issue with:
- The exact command that fails
- The Python version (`/home/lx/miniconda3/envs/snn/bin/python --version`)
- The relevant log file path
